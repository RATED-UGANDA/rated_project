## Bugs found and fixed (post-implementation fixes)

### Image thumbnails only showed placeholders
- **Symptom:** Landing page cards showed colored placeholder blocks even for articles that had real cover images (journalist-uploaded or Pexels-sourced).
- **Root cause:** The frontend ran on `localhost:5173` while backend static files were served from `localhost:5000`. Helmet's default `Cross-Origin-Resource-Policy: same-origin` header blocked the browser from loading cross-origin images, triggering the `onError` fallback to placeholder.
- **Fix:** In `backend/src/app.js`, added `Access-Control-Allow-Origin: *` and `Cross-Origin-Resource-Policy: cross-origin` headers to the `/uploads` and `/assets` static mounts. Also updated `frontend/src/api/imageUtils.js` and `ArticleCard.jsx` to resolve relative image URLs to the full backend origin so the `<img>` tags load correctly.
- **Verification:** Playwright headless check confirmed journalist-uploaded and Pexels thumbnails now render as real `<img>` elements, while seeded placeholder-only articles still show category-colored blocks.

### Submitted short article did not appear in editor queue
- **Symptom:** A one-line article submitted by a journalist was not visible in the editor queue.
- **Root cause:** The LLM validation service has a pre-filter that auto-rejects content shorter than 50 characters. The article went straight to `status='rejected'` and never reached `pending_review` / `llm_checked=true`, so the editor queue filter excluded it.
- **Fix:** This is expected behavior per the project spec — the LLM acts as a gatekeeper against empty/short/non-news content. Verified that a full-length article (200+ characters) submits successfully, becomes `llm_checked=true`, and appears in the editor queue.
- **Verification:** Manual end-to-end test: journalist creates long article → submits → editor queue shows it → editor approves → article appears on homepage with thumbnail.

### 1. Seeded passwords used incompatible bcrypt hash
- **Cause:** `seed.sql` used a Laravel-style `$2b$10$92IXUNpk...` test hash that `bcryptjs` could not verify.
- **Fix:** Replaced all seeded password hashes with hashes generated by `bcryptjs.hashSync(password, 10)`.
- **Verification:** `smoke.js` admin/journalist/editor logins now pass.

### 2. `LIMIT ? OFFSET ?` parameter binding error in article list
- **Cause:** MySQL2's prepared statement binding does not accept numbers for `LIMIT`/`OFFSET` in some versions.
- **Fix:** Cast `limit` and `offset` to integers and interpolate them directly into the query string (safe because values are locally controlled).
- **Verification:** Public article list and homepage load correctly.

### 3. Category/district `admin_id` foreign key violation
- **Cause:** Controller used `req.user.user_id` (the `users` PK) as `admin_id`, but the FK references `admin.admin_id`.
- **Fix:** `category.controller.js` and `district.controller.js` now look up `admin.admin_id` by `user_id` before inserting.
- **Verification:** Plain admin can create categories and districts.

### 4. LLM response parsing and timeouts
- **Cause:** Ollama Cloud sometimes returns partial JSON or the `deepseek-v4-flash` model is slow, causing the 8s timeout to fire.
- **Fix:** Increased timeout to 30s and added a fallback regex JSON extractor in `llmValidation.service.js`. Fail-open behavior preserved.
- **Verification:** Article submissions and scraper validation succeed even when the model is slow.

## Known limitations / demo workarounds

1. **Only one active RSS source.** Of the six configured Ugandan outlets, only **The Independent** exposes a working public RSS feed at build time. The other five are marked `is_active=FALSE` with notes explaining why (Cloudflare bot protection, no feed found, or feed endpoint returns HTML). The scraper still works because it safely skips inactive sources.
   - *Workaround for demo:* The scraper page shows one active source and will fetch fresh articles from The Independent. If the network blocks outbound RSS access, the scraper run will return `fetched: 0` and a clear per-source error.

2. **LLM provider latency.** Ollama Cloud `deepseek-v4-flash` can take 10–25 seconds per call. During heavy load the request may abort and fall back to "forward for manual review".
   - *Workaround:* Article submissions still reach the editor queue because the service fails open. For a snappier demo, a faster/local Ollama model can be set via `LLM_API_URL` and `LLM_MODEL`.

3. **LLM rewrite of scraped content sometimes returns invalid JSON or truncated JSON.** The rewrite service does not require JSON, but the validation service does. The regex fallback extractor handles most cases; rare truncation may still cause fail-open.
   - *Workaround:* Fail-open means the article still enters the editor queue for human review.

4. **Pexels stock images are generic.** Search terms are Uganda-flavored but Pexels does not always return locally specific photography.
   - *Workaround:* The placeholder image is always available and articles display category-colored blocks when no image exists.

5. **Team commit requirement ignored.** Per user instruction, commits were not performed. The repo is ready for the team to commit under their own names.
